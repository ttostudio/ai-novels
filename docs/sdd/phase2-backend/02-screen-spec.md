# 画面仕様書 — Phase 2: バックエンド構築・DB移行・API化

## 概要

Phase 2 はバックエンド API 追加が主目的であり、**既存画面の見た目は変えない**。
変更点は以下の3領域:

1. データ取得方法の変更（JSON直読み → API fetch）
2. 検索UIの新規追加（ヘッダー検索バー + 検索結果表示）
3. ローディング・エラー状態の追加（API通信に対応）

既存デザイントークン（カラー、フォント、スペーシング）は変更なし。

## SCR-004 章リーダー（`/novel/[slug]/[chapter]`）

### 変更点

- `getChapterByNumber()` を `GET /api/novels/{slug}/chapters/{number}` に切替
- **ブックマーク**: `useBookmark` フックを API 呼び出しに変更
  - 追加: `POST /api/bookmarks` (body: `{ novel_slug, chapter_number }`, `X-Session-ID` ヘッダー)

- **PV記録**: `useAnalytics` フックを API 呼び出しに変更

- ページロード中: テキストエリアのスケルトン表示

## SCR-007 検索結果（`/search?q=xxx`）【新規】

### 概要

ヘッダーの検索バーから `GET /api/novels/search?q={query}` を呼び出し、
結果を一覧表示する。

### ヘッダー検索バー（Header.tsx 変更）

```
┌──────────────────────────────────────────────────────────────┐
│  [🔍 AI Novels]  ジャンル▼  [　　検索バー 🔍　　]  📌  📊    │
└──────────────────────────────────────────────────────────────┘
```

| 要素 | 仕様 |
|------|------|
| 配置 | ナビゲーション中央（ジャンルリンクと📌の間） |
| 幅 | デスクトップ: 240px / タブレット: 180px / モバイル: アイコンのみ表示、タップで展開 |
| プレースホルダー | `小説・章を検索...` |
| 入力フォーム | `<form action="/search">` + `<input name="q">` |
| Enter or 検索ボタン | `/search?q={入力値}` へ遷移 |
| デザイン | border: `1px solid var(--border)` / bg: `var(--panel)` / border-radius: `6px` / padding: `6px 12px` |

#### モバイル展開状態

- 検索アイコン（🔍）タップ → 入力フォームが横にスライドイン表示
- フォーカスアウト or Escape → 収納
- アニメーション: `transition: width 0.2s ease`

### 検索結果ページ（`/search?q=xxx`）

```
┌──────────────────────────────────────────┐
│ Header（検索バーに入力値を保持）            │
├──────────────────────────────────────────┤
│ 「"xxx" の検索結果（N件）」                │
├──────────────────────────────────────────┤
│ [検索結果タブ: 小説 | 章]                  │
├──────────────────────────────────────────┤
│ ── 小説タブ ──                            │
│ NovelCard グリッド（マッチした小説）        │
│                                           │
│ ※ 章レベル検索は Phase 3 で対応           │
```

## ローディング状態の共通仕様

### スケルトンコンポーネント

**適用場所**: 全API呼び出しの待機中

| 要素 | スケルトン実装 |
|------|--------------|
| NovelCard | カードと同サイズ（h-48程度）のグレーブロック |
| ChapterList 行 | 高さ 48px、幅 100% のグレーバー |
| 分析チャート | チャートエリアと同サイズのグレーブロック |
| ブックマーク行 | 高さ 64px、幅 100% のグレーバー |

**スタイル仕様**:

```css
.skeleton {
  background: var(--border);         /* #e8e3dd */
  border-radius: 6px;
  animation: pulse 1.5s ease-in-out infinite;
}

@keyframes pulse {
  0%, 100% { opacity: 1; }
  50%       { opacity: 0.5; }
}
```

**表示条件**: API レスポンスが返るまで表示。レスポンス後は実データに即時切替（フェードなし）。

## エラー状態の共通仕様

### エラーバナー

**適用場所**: API呼び出し失敗時（ネットワークエラー、5xx）

```
┌─────────────────────────────────────────────────────┐
│ ⚠️  データの取得に失敗しました。[再試行する →]         │
└─────────────────────────────────────────────────────┘
```

| 要素 | 仕様 |
|------|------|
| 背景色 | `#fff3cd`（警告色、CSS変数外） |
| テキスト色 | `#664d03`（警告色、CSS変数外） |
| border | `1px solid #ffc107` |
| border-radius | `6px` |
| padding | `12px 16px` |
| 「再試行する」 | クリックで同一APIを再fetch |
| 配置 | コンテンツエリア上部 |

### フォールバック表示

| 画面 | フォールバック内容 |
|------|-----------------|
| ホーム | エラーバナー + 「小説一覧を取得できませんでした」 |
| 小説詳細 | エラーバナー + 「この小説の情報を取得できませんでした」 |
| 章リーダー | エラーバナー + 「章の本文を取得できませんでした」 |
| ブックマーク | エラーバナー + 空状態UI（ブックマークなしと同じ表示） |
| 分析 | エラーバナー + 数値を「—」で代替表示 |
| 検索 | エラーバナー + 「検索できませんでした。時間をおいて再試行してください。」 |

## セッションID管理

ブックマーク・分析APIは認証なしで `session_id` を使用する。

| 項目 | 仕様 |
|------|------|
| 生成 | `crypto.randomUUID()` |
| 保存場所 | `localStorage["ai-novels-session-id"]` |
| 初回アクセス時 | 未存在なら生成して保存 |
| UI表示 | なし（ユーザーに見せない） |
| リセット | 明示的なリセット手段なし（Phase 2 スコープ外） |
