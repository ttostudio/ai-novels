# 要件定義書

## 概要

AI Novels プラットフォームに読者分析ダッシュボード（`/analytics`）を追加し、PV数・読了率の可視化を行う。あわせて「次の話へ」ナビゲーションに次章タイトル・冒頭プレビューを表示し、章間離脱を抑制する。

## 背景

- 現在、各章の閲覧数・読了状況を把握する手段がない
- 読者が次章へ進む動機付けが薄く、離脱が発生している
- SSG（静的サイト生成）のため、サーバーサイド集計は不要。localStorage を活用した端末内可視化で十分
- 3作品の一部章でイラストが未生成のため、コンテンツ完成度を高める

## 機能要件

| ID | 要件 | 優先度 | 備考 |
|----|------|--------|------|
| FR-001 | 各章ページにアクセスした際、`novelSlug + chapterNumber` をキーとして localStorage にPV数をインクリメント保存する | Must | ページロード時に1回のみカウント |
| FR-002 | 各章ページで本文末尾（スクロール到達率 90% 以上）に到達した際、読了フラグを localStorage に保存する | Must | IntersectionObserver を使用 |
| FR-003 | `/analytics` ページを新規作成し、作品別・章別の PV 数一覧を表示する | Must | クライアントサイドレンダリング |
| FR-004 | `/analytics` ページに人気作品ランキングを PV 数降順で表示する（作品ごとの合計PV） | Must | 3作品を対象 |
| FR-005 | `/analytics` ページに各章の読了率（読了済みフラグ数 / PV 数 × 100）をパーセント表示する | Must | 小数点1桁 |
| FR-006 | `ChapterNavigation.tsx` を改善し、「次の話へ」ボタンに次章タイトルと冒頭テキスト（最大80文字）をプレビュー表示する | Must | 静的データから取得。Props: `nextChapter?: { title: string; previewText: string }` |
| FR-007 | `/analytics` ページのデータはリアルタイムで localStorage から読み込み、ページ再読み込み不要で最新値を反映する | Should | `useEffect` でマウント時に読み込み |
| FR-008 | 星間漂流 ch5、魔法学園クロニクル ch4、日常茶飯事 ch4 のイラストを生成・配置する | Should | ComfyUI Flux.1-schnell。別タスクで対応。本SDDには要件のみ記載 |
| FR-009 | `/analytics` ページへのリンクをヘッダーナビゲーションに追加する | Should | `Header.tsx` に追記 |
| FR-010 | localStorage データが存在しない（初回訪問）場合、ダッシュボードに「まだデータがありません」メッセージを表示する | Must | エラーではなく正常状態として扱う |

## 非機能要件

| ID | カテゴリ | 要件 | 基準値 |
|----|----------|------|--------|
| NFR-001 | パフォーマンス | `/analytics` ページの初期表示速度 | LCP 2秒以内（ローカルデータのみのため問題なし） |
| NFR-002 | セキュリティ | localStorage の読み書きは自端末内のみ。個人情報・サーバー送信なし | XSS: `JSON.parse` の入力値サニタイズ必須 |
| NFR-003 | 可用性 | localStorage が利用不可（プライベートモード等）の場合、エラーなく動作する | try/catch でフォールバック（PVカウントをスキップ） |
| NFR-004 | アクセシビリティ | ダッシュボードのグラフ・ランキングに ARIA ラベルを付与する | WCAG 2.1 AA 準拠 |
| NFR-005 | レスポンシブ | `/analytics` ページはモバイル（320px〜）でも崩れず表示される | SP/タブレット/PC の3ブレークポイント |

## スコープ

### In（対象範囲）
- `/analytics` ページ新規作成（PV数・読了率・人気ランキング）
- 章ページへのPVカウント・読了率トラッキング埋め込み（localStorage）
- `ChapterNavigation.tsx` の次章プレビュー表示改善
- `Header.tsx` へのアナリティクスリンク追加
- 3章分のイラスト要件定義（実装は別タスク）

### Out（対象外）
- サーバーサイドでの集計・永続化（SSG のため不要）
- 複数端末間のデータ同期
- Google Analytics 等外部トラッキングツール連携
- 管理者専用認証・アクセス制限（URLを知っていれば誰でも閲覧可能）
- 実際のイラスト生成作業（別タスク）

## 用語定義

| 用語 | 定義 |
|------|------|
| PV数 | ページビュー数。同一ページを複数回訪問した場合も都度カウント |
| 読了フラグ | 本文末尾（スクロール到達率90%以上）に到達したことを示す boolean 値 |
| 読了率 | `読了済みセッション数 / PV数 × 100`（%） |
| novelSlug | 作品を識別する文字列（例: `stellar-drift`, `magic-academy`, `daily-life`） |
| SSG | Static Site Generation。ビルド時にHTMLを生成し、サーバーサイドAPIを使わない方式 |

## リスク評価

| ID | リスク | 影響度 | 発生確率 | 対策 |
|----|--------|--------|----------|------|
| R-001 | プライベートモードや設定でlocalStorageが無効な場合、PVカウントが機能しない | 低 | 中 | try/catch でカウントをスキップ。エラー表示なし |
| R-002 | localStorage の容量上限（5MB）超過 | 低 | 低 | 3作品×最大5章程度のキーのみ保存。容量問題は発生しない |
| R-003 | `ChapterNavigation.tsx` の冒頭テキスト取得で静的データが未整備の場合、プレビューが空になる | 中 | 低 | プレビューが存在しない場合はタイトルのみ表示にフォールバック |
| R-004 | イラスト未生成のまま章ページが公開されており、UXが低下している | 中 | 高（既知の課題） | 本タスクで別途イラスト生成を依頼。章ページにプレースホルダーを表示 |

## ロールバック方針

- 切り戻し条件: `/analytics` ページの表示エラー、既存章ページのPVトラッキングコード起因のクラッシュ、ChapterNavigation のリグレッション
- 切り戻し手順:
  1. 問題発生PR を `git revert` してマージ
  2. Docker Compose で再ビルド: `docker compose up -d --build frontend`
  3. localStorage のキーが不正な場合: ブラウザのdevtools > Application > Storage > Clear Site Data
- 影響範囲: フロントエンドのみ（SSG静的ファイル）。データベース・バックエンドへの影響なし

## 受入基準（AC）

| AC-ID | 条件 | 確認方法 |
|-------|------|----------|
| AC-001 | 章ページにアクセスすると、localStorage の `ai-novels:pv` キーにPVデータが記録される | DevTools > Application > localStorage でキー確認 |
| AC-002 | 本文を90%以上スクロールすると、localStorage の `ai-novels:reading-progress` キーに読了データが保存される | DevTools > Application > localStorage でキー確認 |
| AC-003 | `/analytics` ページが HTTP 200 で表示され、PV 数・人気ランキング・読了率の3セクションが全て描画される | ブラウザで URL にアクセス、目視確認 |
| AC-004 | 人気作品ランキングが PV 数降順（最大PV作品が1位）で表示される | localStorage にダミーデータを設定後、ランキング順序を確認 |
| AC-005 | 読了率が小数点1桁のパーセント形式（例: 75.0%）で各章に表示される | localStorage にPV=4・読了=3 のダミーデータを設定し、75.0% と表示されることを確認 |
| AC-006 | localStorage にデータが存在しない場合、`/analytics` ページに「まだデータがありません」メッセージが表示される | localStorage を空にした状態でアクセス |
| AC-007 | `ChapterNavigation.tsx` の「次の話へ」ボタンに次章タイトルが表示される | 章ページ末尾のナビゲーション目視確認 |
| AC-008 | 「次の話へ」ボタンに次章冒頭テキストが80文字以内でプレビュー表示される | 章ページ末尾のナビゲーション目視確認（文字数カウント） |
| AC-009 | localStorage 無効環境（プライベートモード相当）でも章ページがエラーなく動作する | try/catch 実装の確認（コードレビュー） |
| AC-010 | `/analytics` ページがモバイル（320px幅）で崩れず表示される | Chrome DevTools のモバイルシミュレーターで確認 |
